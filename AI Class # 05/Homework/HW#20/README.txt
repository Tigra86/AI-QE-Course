--------------------------------------------------------------------------------
1. PROJECT OVERVIEW
--------------------------------------------------------------------------------
The goal of this assignment was to analyze a series of chatbot interactions 
using a Reinforcement Learning framework. Each scenario evaluates a specific 
chatbot "action" and assigns a "reward" score based on the quality and 
helpfulness of the response.

--------------------------------------------------------------------------------
2. SCENARIO REPOSITORY
--------------------------------------------------------------------------------
- PASSWORD RESET (rl_scenario_a.json):
  * Action: Provide clear password reset instructions.
  * Reward: 1.8 (Positive experience due to accurate, direct guidance).

- OUT-OF-SCOPE REJECTION (rl_scenario_b.json):
  * Action: Rejection of a weather query.
  * Reward: -0.5 (Penalty for a blunt rejection without helpful redirection).

- STOCK STATUS (rl_scenario_c.json):
  * Action: Provide precise and timely product availability.
  * Reward: 2.0 (High reward for helping the user make a quick decision).

- TROUBLESHOOTING (rl_scenario_d.json):
  * Action: Request for clarification and error details.
  * Reward: 1.0 (Neutral/Positive for attempting to diagnose before answering).

- HUMAN ESCALATION (rl_scenario_e.json):
  * Action: Successful transfer to a human representative.
  * Reward: 2.5 (Highest reward for recognizing clear intent and immediate action).

--------------------------------------------------------------------------------
3. DATA SCHEMA
--------------------------------------------------------------------------------
To ensure consistency for the RL agent, every scenario file contains:
- USER_MESSAGE: The raw input from the user.
- CHATBOT_RESPONSE: The text output generated by the model.
- ACTION: The specific category of behavior being tested.
- REWARD: The numeric value used to train the model's policy.
- REASON: The justification for the assigned reward score.

--------------------------------------------------------------------------------
4. USE CASE
--------------------------------------------------------------------------------
These JSON files serve as the "labeled data" for an RL agent. By processing 
these rewards, the agent learns to prioritize helpful actions (like 
escalation) over unhelpful ones (like blunt rejections) in future 
conversations.
